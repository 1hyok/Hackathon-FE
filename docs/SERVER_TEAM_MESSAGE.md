# 서버 팀에게 전달할 메시지

안녕하세요! 안드로이드 팀입니다.

API 명세서에 대한 상세한 Request/Response 구조를 정리했습니다.

## 📄 문서 위치

프로젝트 루트의 `docs/API_SPECIFICATION_DETAIL.md` 파일에 모든 내용을 정리해두었습니다.

## 📋 포함된 내용

각 API 엔드포인트에 대해 다음 정보를 포함했습니다:

1. **Request Body 구조** (JSON 형식)
   - 필수/선택 필드 명시
   - 데이터 타입 및 제약사항

2. **Response Body 구조** (JSON 형식)
   - 성공 응답 예시
   - 에러 응답 예시

3. **헤더 정보**
   - 인증 토큰 사용 여부
   - Content-Type 등

4. **화면별 필요한 데이터**
   - 각 화면에서 실제로 사용하는 필드들

## 🔍 주요 사항

### 1. 공통 Response 구조
모든 API는 다음 형식을 따릅니다:
```json
{
  "code": 200,
  "message": "성공 메시지",
  "data": { /* 실제 데이터 */ }
}
```

### 2. 인증
- JWT 토큰을 `Authorization: Bearer {token}` 형식으로 전달
- 대부분의 API에서 인증 필요

### 3. 이미지 업로드
레시피 등록 시 이미지를 함께 업로드하는 방식과 별도 API로 분리하는 방식 중 선택해주시면 감사하겠습니다.

### 4. 페이지네이션
목록 조회 API는 페이지네이션 지원이 필요합니다.
- 기본값: `page=0`, `size=20`

### 5. 좋아요 기능
목록 조회 시 현재 사용자가 좋아요를 눌렀는지 여부(`isLiked`)를 포함해주시면 좋겠습니다.

## ❓ 질문이나 확인이 필요한 사항

1. **이미지 업로드 방식**: multipart/form-data로 함께 보낼지, 별도 API로 분리할지
2. **카테고리 값**: 정확한 카테고리 값 확인 (현재는 '서브웨이', '하이디라오', '편의점'으로 가정)
3. **재료 데이터 구조**: 목록에서는 문자열 배열, 상세에서는 객체 배열로 구분할지
4. **날짜 형식**: ISO 8601 형식 또는 Unix timestamp 중 선택
